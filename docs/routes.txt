Маршрутизация запросов

каталог routing
- web
- api
- console
- channels

Деление запросов на группы и задание маршрутов для них.
Деление происходит по некоторым особенностям запросов.
Например для апи запросов, зачастую необходима авторизация и тд.
Идёт поиск первого подходящего маршрута и запуск действий прописанных ему.
Если нет подходящего то отдаёт 404.

web.php
get(string $uri, array|string|callable|null $action = null)
- 1-ый параметр адрес по которому был совершён веб запрос
- 2-ой действия которые необходимо совершить
возвращает строку, файл
В фун-ии обратного вызова можно задавать доп логику, но принято это делать в контроллере


Каждому http запросу соответствует одноимённый метод класа Route 

Route::get($uri, $callback);
Route::post($uri, $callback);
Route::put($uri, $callback);
Route::patch($uri, $callback);
Route::delete($uri, $callback);
Route::options($uri, $callback);

put и delete доработанный post, для соответствующих действий
