Паттерн mvc
model - структура в которой представленны данные, полученные по запросу к хранилищу
view - конечное состояние данных, выводимое пользаку
controller - принимает запрос и работает с конкретной моделью и представлением

Пример
Пришли в магазин за чаем
model чай
controller продавец, собирает необходимое кол-во в упаковку
представление упаковка с чаем


Структура laravel
app - ядро приложения
- console классы инструменты artisan
- Exceptions обработчики исключений
- Providers сервис провайдеры,
- http находятся контроллеры и посредники
- модели лежат в корне папки app

bootstrap - файлы фреймворка
- cache кеш файлов фреймворка

config - файлы настроек

database - всё что касается бд, миграции, файлы работы с бд

public - публичная часть сайта в том числе js скрипты стили  тд

resourses - не скомпилированные файлы стилей s скриптов view lang файлы и тд

routes - папка с маршрутами.
В каждом файле прописаны маршруты для определённой группы запросов, api web и тд
- маршруты по которым пойдут запросы

storage - скомпилированные шаблоны blade и другое

test - автотесты

vendor - автозазагрузка и сторонние библиотеки загруженные composer ом
